07
Strategic Insights & Management Notes

Observations and recommendations for cost governance, optimisation, and planning.

🔴 Cost Priority

Extraction Pipeline Drives 78% of Cost

Document extraction (1,106 calls) is the dominant spend. Each additional run adds ~₹3,250–3,750. Implementing document caching and change-detection before re-extraction would be the highest-ROI optimisation available.

📐 Architecture

Thinking Tokens = Quality Investment

49% of all tokens are thinking tokens. This is by design — the model reasons internally before answering complex tasks. Disabling thinking would reduce costs ~40% but risks accuracy in price matching and specification disambiguation.

✅ Efficiency

MTO Pipeline is Highly Cost-Efficient

At <1% of total spend (₹60–65 per run), MTO generation delivers disproportionate value. The cost per priced MTO line item is negligible. No optimisation needed in this pipeline.

📋 Scalability

SOR Cost Scales Linearly with PO Volume

The SOR Price Search LLM cost grows proportionally with PO line items (400 items = ~₹1,500–1,900). For every 100 additional PO items, expect ~₹375–475 in additional SOR pipeline cost.

💡 Optimisation

Embedding Calls are Effectively Free

All embedding calls combined (112 total) cost ₹0.23 — negligible. Embedding-first search architecture is the correct approach; expand semantic search coverage without cost concern.

📅 Projection

Budget Planning for Recurring Runs

Assuming quarterly extraction runs + monthly MTO and SOR runs: projected annual AI API cost is approximately ₹30,000–40,000. Establish a monthly API cost monitoring dashboard to track actuals vs this baseline.

Terminology Reference
Prompt Tokens
Text sent to the model — instructions, document content, and context. Billed as input tokens.
Output Tokens
Text returned by the model. Typically 5–15% of input volume. Billed at output rate.
Thinking Tokens
Internal chain-of-thought reasoning, not shown in output. Improves accuracy for complex tasks. Billed separately.
Embedding Tokens
Input-only tokens used to convert text into search vectors. No output or thinking tokens; extremely cheap.
